# Power Validation

Record measured power behavior here.

Each entry must include:

- date and time
- board revision
- firmware commit
- knobs hash or version
- runtime class or mode
- sleep class
- wake source
- average current
- wake latency
- resume latency
- artifact links

Unknown wake reasons remain defects until explained.
