# Progression Architecture

Progression in the Reference Game should be about familiarity, access, relationship, observation, and strange capability growth.

It is not primarily about content throughput.

## Progression Axes

| Axis | Meaning | Related notes |
|---|---|---|
| relationship | the player and slime understand each other more deeply | [[Slime_Relationship_Model]], [[Pet_Simulation_and_Hidden_State]] |
| forms | the slime gains adaptive identities and social/traversal access | [[Transformation_and_Form_System]], [[Forms_Index]] |
| world knowledge | the player learns schedules, rituals, and hidden patterns | [[Observation_and_Temporal_Ecology]], [[Temporal_World_and_NPC_Routines]] |
| space access | new places open through forms, routines, or tools | [[Places_Index]], [[Turn_Based_Exploration]] |
| ritual literacy | the player learns how to participate in occult events | [[Rituals_Index]], [[Death_and_Resurrection_Ritual]] |
| combat literacy | the player learns enemy timing, positioning, and split-slime tactics | [[Combat_and_Split_Slime]], [[Enemies_Index]] |
| hardware literacy | the player learns physical interactions as fiction | [[Reference_Game_Capability_Map]], [[Realtime_Microgame_Catalog]] |

## Progression Rules

- Major unlocks should teach a new way of seeing the world.
- Forms should change social meaning, not only stats.
- Real-time bursts should punctuate progression, not become the main loop.
- The player should be able to infer some progress by observing the slime and NPCs.
- Hardware interactions should feel like rituals, creature handling, or environmental sensing.
