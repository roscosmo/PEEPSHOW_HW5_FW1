# Reference Game Capability Demonstration Roadmap

This note ensures the Reference Game proves the device without becoming a hardware checklist.

## Demonstration Rule

Every capability should be introduced through fiction, emotion, or play.

Do not introduce a hardware feature only because the hardware exists.

## Capability Debt

| Capability | Design risk | Needed concept |
|---|---|---|
| BLE | may feel bolted on | define fictionally meaningful multiplayer or companion-app use |
| step counter | could feel arbitrary | define carrying/walking behavior that affects slime without becoming a chore |
| charging/battery | could feel like maintenance | define care language and low-battery mood expression |
