## Decision Rules

- Prefer decisions that preserve the illusion of a living artifact.
- Prefer systems that create observation, memory, and ritual.
- Reject mechanics that turn care into routine maintenance unless that friction is intentional.
- Reject hardware use that feels like a checklist instead of part of the fiction.
- If a decision needs reusable behavior, update Engine docs.
- If a decision needs hardware policy, update Platform docs first.
