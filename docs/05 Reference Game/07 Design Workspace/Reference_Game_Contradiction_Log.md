# Reference Game Contradiction Log

This note tracks unresolved design tensions.

Contradictions are useful while designing. They only become problems when implementation starts without deciding which side owns the rule.

## Resolution Pattern

For each contradiction:

1. Decide the emotional priority.
2. Decide the player-facing rule.
3. Decide the hidden system rule.
4. Link the system or content note that owns it.
5. Record the decision in [[Reference_Game_Decision_Log]].
