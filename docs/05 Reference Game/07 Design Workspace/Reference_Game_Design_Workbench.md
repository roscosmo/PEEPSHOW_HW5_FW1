# Reference Game Design Workbench

This is the working desk for developing the Reference Game.

Use this area for ideas that are still being shaped, compared, questioned, or connected. Finished decisions should eventually flow into the relevant design note, system note, or content bible entry.

## Working Flow

1. Capture loose ideas in [[Reference_Game_Idea_Inbox]].
2. Promote useful ideas into specific notes using [[Design_Element_Template]].
3. Record locked or rejected decisions in [[Reference_Game_Decision_Log]].
4. Track unresolved design tension in [[Reference_Game_Contradiction_Log]].
5. Convert capability needs into [[Reference_Game_Capability_Map]] entries.
6. Move stable content into [[Content_Bible_Index]] or [[Progression_Architecture]].

## Design Maturity

Use these statuses inside notes:

| Status | Meaning |
|---|---|
| `seed` | interesting idea, not yet connected |
| `sketch` | rough shape exists, still volatile |
| `candidate` | likely useful, needs pressure testing |
| `committed` | accepted design direction |
| `cut` | rejected or deferred |

## Core Working Questions

- Does this idea strengthen [[Living_Artifact_and_Persistence]]?
- Does it support [[Observation_and_Temporal_Ecology]]?
- Does it deepen [[Slime_Relationship_Model]]?
- Does it fit one or more runtime modes in [[Reference_Game_Runtime_Mode_Map]]?
- Does it demonstrate a Platform or Engine capability through [[Reference_Game_Capability_Map]]?
- Does it stay inside the boundary in [[Game_Documentation_Boundary]]?
