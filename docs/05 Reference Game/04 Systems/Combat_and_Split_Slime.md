# Combat And Split Slime

Combat is fundamentally turn-based, tactical, and positional.

Realtime bursts may punctuate combat, but they do not replace the turn-based core.

## Turn-Based Combat Core

The player may:

- move
- attack
- use forms
- split the slime
- trigger specials
- reposition tactically
- interact with terrain

Enemies may:

- react
- reposition
- attack
- trigger events
- force realtime microgames

## Presentation Modes

Combat presentation may:

- stay in the tile view
- zoom in
- show side-view animations
- use first-person attacks
- trigger a [[Realtime_Microgame_Runtime]] burst

## Split Slime System

Normally the slime follows the player.

The player may separate the slime for:

- dual-character movement
- scouting
- flanking
- puzzle solving
- baiting enemies
- distant interaction

Distance may affect:

- slime stability
- emotional behavior
- control reliability
- vulnerability
- relationship tension

## Design Rule

Splitting the slime should be tactically useful but emotionally charged.

The slime is not a disposable unit.
