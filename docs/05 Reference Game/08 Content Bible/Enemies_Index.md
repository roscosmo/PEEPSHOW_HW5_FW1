## Enemy Entry Requirements

Each enemy note should define:

- overworld or dungeon context
- turn-based movement rule
- attack rule
- tell or warning language
- realtime microgame, if any
- first-person portrait need, if any
- slime form interactions
- reward, consequence, or lesson
