# Reference Game Design Pillars

## Living Artifact

The hardware is part of the fiction.

The shell, texture, persistent display, wheel, battery behavior, sensors, and sleep behavior should all support the belief that the creature physically exists inside the object.

See [[Living_Artifact_and_Persistence]].

## Persistence Creates Attachment

If the creature disappears, the player subconsciously feels it stopped existing.

If it remains visible, the player feels it continues living independently.

Persistent visual presence is central to the emotional design.

## Observation Is Gameplay

Observation is not downtime.

The player watches routines, notices patterns, follows suspicious movement, detects environmental changes, and learns hidden systems.

See [[Observation_and_Temporal_Ecology]].

## The World Acts Without Permission

NPCs and systems move, gather, disappear, travel, and perform rituals without direct player prompting.

The game should create the sense that things are happening even when the player is not looking.

## Time Is A Game System

Real-world time controls routines, events, rituals, tendencies, social behavior, and rare moments.

This is a design need expressed to the Engine and Platform through [[Runtime_Host_Contract]] and [[Power_and_Sleep_Policy]].

## Genre Expectations Are Mutated

The game sincerely understands virtual pets, eggs, evolution, care, and ritual expectations.

It then mutates those expectations into stranger creature logic.

The player should repeatedly feel: "I thought I understood this. I was wrong."

## Relationship Over Completion

The game is not built around completion speed, content throughput, or optimization loops.

It is built around memory, atmosphere, ritual, companionship, curiosity, emotional continuity, and long-term familiarity.
