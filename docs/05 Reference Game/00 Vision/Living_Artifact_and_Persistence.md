# Living Artifact And Persistence

The Reference Game treats the device as a physical vessel for the slime.

The player should feel that the slime is carried, checked on, cared for, and lived beside.

## Always-Visible Presence

The [[Display_and_Rendering_Contract]] makes the always-visible Sharp Memory LCD emotionally important.

The game should use persistent display state to imply continued existence:

- idle posture
- sleep posture
- blinking
- small dreams
- wobble and breathing-like motion
- ambient reactions
- low-battery posture
- charging comfort
- time-of-day behavior

## Absence Has Meaning

If the display goes still after a major narrative event, the stillness should mean something.

The strongest use of the persistent display may be the rare moment when it no longer behaves like a living window.

See [[Death_and_Resurrection_Ritual]].

## Device Body As Fiction

Physical affordances are part of the relationship:

- rotary encoder as petting, scratching, winding, and ritual tracing
- light sensor as darkness, exposure, and hiding behavior
- IMU as orientation, handling, movement habits, and balance
- battery as emotional care pressure
- audio as tiny creature vocalization and ritual sound

These are capability requests summarized in [[Reference_Game_Capability_Map]].

## Design Rule

The device should not feel like a neutral console showing a game.

It should feel like a strange container holding a creature.
