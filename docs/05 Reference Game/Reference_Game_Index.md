# Reference Game Index

The Reference Game is the PeepShow proof-of-capability game.

It is a real game, not a throwaway demo. It exists to prove the device, engine, asset pipeline, and development tools while remaining a consumer of those layers.

The game should feel like a living fictional organism inhabiting a physical handheld artifact.

## Boundary

The Reference Game may define:

- creatures
- world design
- narrative
- emotional design
- mechanics
- balancing
- encounters
- progression
- UI flow intent
- game-specific FSMs
- capability demonstrations

The Reference Game may not define:

- Platform hardware behavior
- peripheral ownership
- RTOS topology
- DMA policy
- storage ownership
- sleep policy
- Engine API rules that should apply to all games
