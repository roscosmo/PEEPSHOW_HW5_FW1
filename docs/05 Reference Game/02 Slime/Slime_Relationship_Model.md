# Slime Relationship Model

The relationship with the slime is the narrative core of the Reference Game.

The main emotional question is not whether the player maximizes stats.

The main emotional question is whether emotional continuity with the slime is maintained.

## Relationship Goals

The relationship should:

- deepen
- strain
- recover
- become vulnerable
- create shared rituals
- produce memory through repeated small interactions
- change how the slime reacts to the player

## Shared Rituals

The game should create moments that feel shared:

- late-night discoveries
- repeated check-ins
- petting or scratching routines
- ritual attendance
- a familiar response after a hard event
- a tiny behavior that only happens after trust builds

## Vulnerability

Eventually, the player should fear:

"What if I lost my friend?"

Not merely:

"What if I lose progress?"

## Engine Needs

The relationship model likely needs:

- persistent relationship variables
- routine memories
- mood and trust state
- event flags
- form history
- ritual participation history
- save migration support through [[Package_Contract]]
